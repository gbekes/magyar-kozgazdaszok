# CLAUDE.md — project-specific instructions for Claude Code

## What this project is

**Magyar Közgazdászok** — a curated catalogue of peer-reviewed research by top
Hungarian economists (and diaspora), framed for Hungarian policymakers. Static
site: plain HTML + vanilla JS + CSS, no framework, no build step for pages.
JSON files are the data layer. Deploys via GitHub Pages on push to `main`.
Editor: Gábor Békés.

- `index.html`, `papers.html`, `paper.html`, `authors.html`, `author.html`,
  `topics.html`, `topic.html`, `about.html`, `contribute.html`, `journals.html`

## Conventions

- Every substantive change gets a commit message explaining *why*, with a
  `Co-Authored-By: Claude Opus 4.7 (1M context) <noreply@anthropic.com>` footer.
- Don't push unless the user says "push" (or equivalent).
- Run `python build.py` after any `data/` change before committing.
- UI string changes: update both EN and HU where applicable.
- In-language nav uses bare relative links (no `../` prefix); only the
  language toggle crosses the `/hu/` boundary.
- `rootPrefix()` in `assets/app.js` returns `''` — don't re-add the old `../`
  behavior, it causes HU→EN leakage on every internal link.

## Author-submission workflow

Authors email free-text corrections to the editor. Editor drops them into
`submissions/YYYY-MM-DD-<surname>.md` and asks Claude to process. Claude:

- Reads each file, applies changes to the right JSON fields.
- Auto-flips `review_status` to `author-approved` for affected entries.
- One commit per submission, provenance in the message.
- Moves file to `submissions/DONE/` (gitignored).
- Flags (no commit) on: new-author admission, ambiguous targets,
  destructive changes, taxonomy changes, contradictory edits.

Full rules in `submissions/README.md`.

## Session-specific quirks worth knowing

- **OpenAlex author clusters** have frequent namesake issues for common
  Hungarian surnames. Already cleaned out false positives for:
  - Attila Lindner — OpenAlex conflated him with Stefano DellaVigna's
    co-author cluster. Most `lindner-dellavigna-*` papers from 2015–2020
    are actually DellaVigna's sole/Malmendier work. Keep:
    `dellavigna-lindner-et-al-2017-qje` (real, Hungary minimum wage) and
    the 2025 REStud / WP (real recent collaboration).
  - Balázs Reizer — same cluster as Lindner. Direct OpenAlex search for
    him returns almost nothing; his real papers show up through Lindner's
    / Card's co-author networks.
  - Ferenc Szűcs — OpenAlex cluster is a pharmacology namesake. Add his
    real papers manually from his website.
  - Dániel Horn — Obesity-journal researcher shares the name.
  - László Sándor — nuclear physicist CMS-experiment namesake.
  - Tamás Fleiner / Zsuzsanna Jankó — real economists, but the user
    chose to drop them as too theory/math-adjacent for the catalogue.
    Don't re-add unless explicitly asked.

- **`ingest.py discover`** accepts `--since-year N` and
  `--sort publication_year:desc|cited_by_count:desc`. Default is
  cited-by-count desc. For "all recent work" runs use
  `--since-year 2018 --sort cited_by_count:desc`. Default
  `--max-works` is too low for prolific authors — use 50+ to avoid
  under-discovery (we've hit this repeatedly).

- **RePEc is the cleaner source for WP series.** OpenAlex rarely
  identifies CEPR DPs / IZA DPs / CESifo WPs as distinct locations —
  they hide inside SSRN records. RePEc (ideas.repec.org/e/<id>.html)
  lists them explicitly with DP numbers. When available, store the
  `repec_id` on the author JSON and use the RePEc page for spot-check
  and for filling in missed CEPR/IZA WPs.

- **Editor's WP rule (catalogue-wide):** a working paper is kept
  only if `year ≥ 2023` AND it's in CEPR / NBER / IZA / CESifo. In
  practice NBER is the only one that's reliably detectable from
  OpenAlex metadata (DOI `10.3386/w…`). Everything else that's
  ambiguous should be deleted; the editor can re-add from a named
  RePEc WP series by hand. Published articles in peer-reviewed econ/
  finance journals are kept regardless of year.

- **`build.py`** also expands `authors-seed.json` into per-author JSONs
  (only creates files that don't exist — doesn't overwrite).

## Handover rule — context-window management

**When approaching ~88% of context-window max tokens**, proactively create a
`handover-YYYY-MM-DD.md` at the project root, commit it locally (don't push),
and tell the user. A fresh Claude session should be able to read it and
continue the work cold.

The handover file must include, in order:

1. **One-paragraph project summary** — enough for a fresh session to orient.
2. **Current HEAD** — output of `git rev-parse HEAD` plus `git log --oneline -5`.
3. **Working-tree state** — `git status --short`, with a note on why anything
   is uncommitted.
4. **Open TODOs** — TodoWrite items still pending + user-queued items.
5. **Recent session summary** — last 5–10 user instructions and the actions
   taken for each (one line each).
6. **Open questions** — anything the user has flagged for decision but hasn't
   decided on yet.
7. **Session-specific context** — anything learned this session that wouldn't
   be in `CLAUDE.md` or the committed docs, but would bite a fresh session.
8. **Next step** — what you would do first if the user said "continue".

Format: plain markdown, ~1–2 pages. Commit message:
`Handover document — <date>, <reason for handover>`.
