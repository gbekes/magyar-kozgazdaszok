Contribute

Corrections, missing papers, missing authors — all welcome. There are three ways in.

1. Flag a missing paper or correction

Easiest path: email the editor at bekesg@ceu.edu with:

  • Paper title, authors, year, journal, DOI.
  • What's wrong or missing. Short is fine.

If you are the author of a paper on the site and want to adjust how it's summarised, say so — author-approved summaries get a badge.

2. Submit via pull request

If you're comfortable with git, the data lives in JSON files in the repo:

  • One file per paper at data/papers/<slug>.json
  • One file per author at data/authors/<slug>.json

Slug format: firstauthor-lastauthor-year-journalshort (e.g., lindner-harasztosi-2019-aer). Use data/sample-paper.json as a template. Run python build.py locally to regenerate data/index.json, then open a PR.

3. Suggest an author

Inclusion criterion: at least one paper in a top general or top field journal (see the SPEC). Broader "Hungarian" definition: nationality, birthplace, or three or more years primary affiliation. If you think someone qualifies and isn't listed, write in.

How summaries get written

When a paper is added, the summary and policy-relevance fields start as ai-drafted. A human editor reviews, tightens, and flips them to human-reviewed. If the author reads and approves, the status moves to author-approved and a small badge appears on the paper. Authors can submit their own draft at any time.

What we won't accept

  • Op-eds, blog posts, or policy notes without an abstract and a clear working-paper version.
  • Papers that aren't yet publicly posted (submit after posting a working paper).
  • Summaries that copy-paste the abstract — we want plain language.
